# AI 分析日期区间选择功能

## 需求概述

用户生成 AI 分析报告时，可以选择特定日期区间的睡眠记录进行分析，而非固定分析最近 30 条记录。

## 功能设计

### 交互流程

1. 用户在 Dashboard 页面点击"生成 AI 报告"按钮
2. 弹出日期区间选择对话框
3. 对话框显示日历选择器，有记录的日期高亮标记
4. 用户选择开始和结束日期
5. 系统提示区间内无记录的天数（如有）
6. 用户确认后生成报告

### 默认区间逻辑

- **最近连续有记录区间**：从最新一条记录日期开始，向前遍历直到遇到无记录的日期
- 示例：若用户有 3/1-3/10 的记录，3/11 无记录，3/12-3/15 有记录，则默认区间为 3/12-3/15

### 日历显示

- 有睡眠记录的日期显示圆点标记或背景高亮
- 用户只能选择有记录的日期范围（不可选择未来日期）
- 支持月份切换

### 无记录提示

- 选择区间后，计算区间内无记录的天数
- 若存在无记录日期，显示提示："该区间内有 X 天无记录"
- 允许用户继续生成报告（不强制阻止）

## 技术实现

### 前端修改

**文件**: `src/app/dashboard/page.tsx`

- 添加对话框状态管理（open、dateRange）
- 添加日期区间选择器组件（使用 shadcn Calendar + Popover）
- 点击"生成 AI 报告"按钮时打开对话框
- 获取用户所有有记录的日期，在日历上标记
- 计算默认区间（最近连续有记录）
- 计算区间内无记录天数并显示提示
- 确认后调用 API 并传递日期参数

### 后端修改

**文件**: `src/app/api/analyze/route.ts`

- 接收 `startDate` 和 `endDate` 查询参数
- 根据日期范围查询睡眠记录
- 若未传参数，使用默认区间逻辑（与前端一致）
- 返回该区间内的分析报告

### 新增依赖

- `date-fns` — 日期处理库（如未安装）
- shadcn `calendar` 组件 — 已安装
- shadcn `popover` 组件 — 已安装

## 数据结构

### API 请求

```typescript
POST /api/analyze
Body: {
  startDate?: string;  // "2024-03-01"
  endDate?: string;    // "2024-03-15"
}
```

### 日期区间状态

```typescript
interface DateRange {
  from: Date | undefined;
  to: Date | undefined;
}
```

## 边界情况

1. **无任何记录**：不显示对话框，保持现有"暂无数据"提示
2. **区间内全部无记录**：显示错误提示，禁止生成
3. **单日记录**：允许选择单日进行分析
4. **区间过大**：不限制，由 AI 处理（可考虑后续添加上限提示）

## 验收标准

- [ ] 点击按钮弹出日期选择对话框
- [ ] 日历上正确标记有记录的日期
- [ ] 默认选中最近连续有记录的区间
- [ ] 选择区间后正确显示无记录天数提示
- [ ] 生成的报告数据范围与选择区间一致
- [ ] 对话框支持取消操作
