# 研究发现

## 代码库结构
- **框架**: Next.js (有 breaking changes，需参考 `node_modules/next/dist/docs/`)
- **UI 库**: shadcn/ui
- **AI**: 讯飞星火模型 (`src/lib/ai.ts`)
- **数据库**: Prisma + SQLite

---

## 发现的问题

### 1. alert() 使用位置
| 文件 | 行号 | 用途 |
|------|------|------|
| `src/app/page.tsx` | 19 | 文件类型错误提示 |
| `src/app/page.tsx` | 36 | 导入成功提示 |
| `src/app/page.tsx` | 39 | 导入失败提示 |
| `src/app/page.tsx` | 42 | 上传异常提示 |
| `src/app/dashboard/page.tsx` | 63 | 分析失败提示 |

### 2. AI 请求问题
- `src/lib/ai.ts` 无 timeout
- `src/app/api/analyze/route.ts` 无超时包装
- 前端 fetch 无超时，状态可能卡死

### 3. 错误处理问题
- 网络错误和"不存在"混淆
- 部分失败静默跳过
- JSON 解析失败无提示

### 4. 缺失功能
- 无 `error.tsx` 错误边界
- 无专用单报告 API
- 使用 `window.location.href` 而非 `router.push()`

---

## 技术决策
- Toast 使用 `sonner` (shadcn 推荐)
- 超时使用 `AbortController`
- 重试使用指数退避策略
