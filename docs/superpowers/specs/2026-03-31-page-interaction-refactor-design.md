# 页面交互重构设计

## 概述

对首页和历史记录页进行全面交互重构，提升用户体验。

## 首页重构

### 当前问题

1. **按钮布局混乱**：右上角按钮堆叠，主次不分明
2. **跳转体验生硬**：上传成功后直接跳转，无过渡
3. **登录状态切换**：登出后状态更新不流畅

### Header 设计

**已登录状态：**
```
┌─────────────────────────────────────────────────────────┐
│  [Moon] 睡眠分析              [看板] [登出] [主题切换]   │
└─────────────────────────────────────────────────────────┘
```

**未登录状态：**
```
┌─────────────────────────────────────────────────────────┐
│  [Moon] 睡眠分析                        [登录] [主题]    │
└─────────────────────────────────────────────────────────┘
```

### 交互优化

1. **上传成功跳转**
   - 显示成功 toast
   - 1.5 秒延迟后跳转，给用户反应时间
   - 跳转时添加页面过渡效果

2. **登出交互**
   - 登出按钮添加确认提示
   - 登出成功后显示 toast
   - 页面状态平滑过渡

3. **按钮分组**
   - 主要操作：看板（primary 样式）
   - 次要操作：登出（outline 样式）
   - 设置：主题切换（ghost 样式）

### 涉及文件

- `src/app/page.tsx` - 首页主文件

## 历史记录页重构

### 当前问题

1. **筛选器样式不统一**：原生 input 与 shadcn 风格不一致
2. **筛选交互无反馈**：点击筛选后无 loading 状态
3. **表格体验一般**：hover 效果简单

### 筛选区域设计

```
┌─────────────────────────────────────────────────────────┐
│ 筛选条件                                                │
│                                                         │
│ ┌──────────────┐  ┌──────────────┐                     │
│ │ 开始日期     │  │ 结束日期     │                     │
│ │ 2024-01-01   │  │ 2024-01-31   │                     │
│ └──────────────┘  └──────────────┘                     │
│                                                         │
│ [筛选] [清除]                        找到 30 条记录     │
└─────────────────────────────────────────────────────────┘
```

### 交互优化

1. **筛选器样式**
   - 使用 shadcn/ui Input 组件
   - 统一 border、focus、padding 样式
   - 添加日历图标

2. **筛选交互**
   - 筛选按钮添加 loading 状态
   - 筛选后显示结果数量
   - 清除按钮添加确认

3. **表格优化**
   - 行 hover 添加背景色变化
   - 添加行点击高亮效果
   - 评分徽章样式优化

### 涉及文件

- `src/app/history/page.tsx` - 历史记录页主文件
- `src/components/ui/input.tsx` - 需要添加 Input 组件（如不存在）

## 统一组件规范

### 按钮

| 类型 | 样式 | 用途 |
|------|------|------|
| Primary | `variant="default"` | 主要操作（筛选、提交） |
| Outline | `variant="outline"` | 次要操作（清除、登出） |
| Ghost | `variant="ghost"` | 图标按钮（主题切换） |
| Destructive | `variant="destructive"` | 危险操作（删除） |

### 输入框

统一使用 shadcn/ui Input 组件：
```tsx
<Input
  type="date"
  value={startDate}
  onChange={(e) => setStartDate(e.target.value)}
/>
```

### Toast

使用 sonner 统一提示：
- 成功：`toast.success()`
- 错误：`toast.error()`
- 警告：`toast.warning()`
